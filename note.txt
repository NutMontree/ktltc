หากเกิดเหตุการณ์เตือนหรือต้องการอัปเดตโค้ดใหม่ (Deploy/Rebuild) และรีสตาร์ทเซิร์ฟเวอร์อีกครั้ง ให้ใช้ชุดคำสั่งนี้ตามลำดับครับ:

1. ลบไฟล์แคชและไฟล์ล็อคของการ Build (ป้องกันปัญหา lock ชนกัน)
ก่อนจะรัน Build ใหม่ทุกครั้ง แนะนำให้ลบแคชตัวเก่าและปลดล็อคการรันก่อนด้วยคำสั่ง:

bash
# เข้าไปที่โฟลเดอร์โปรเจกต์
cd ~/ktltc
# ลบโฟลเดอร์แคชของ Next.js ตัวเก่าออก
rm -rf .next
# ลบไฟล์ล็อคที่ทำให้เกิดข้อผิดพลาด "Unable to acquire lock"
rm -f .next/lock
2. ทำการ Build โปรเจกต์ใหม่
สั่ง Build เพื่อแปลงไฟล์ TypeScript/Next.js เป็นไฟล์ Production:

bash
npm run build
3. รีสตาร์ทบริการบน PM2 เพื่อเริ่มทำงานด้วยโค้ดใหม่
เมื่อทำการ Build สำเร็จแล้ว ให้ทำการรีสตาร์ทตัวบริการแอปที่รันอยู่ในเบื้องหลัง:

กรณีทั่วไป (รีสตาร์ทตัวเดิมทันที):
bash
pm2 restart ktltc
กรณีที่ต้องการเคลียร์โปรเซสและเริ่มแอปใหม่ตั้งแต่ต้น (แบบสะอาดที่สุด):
bash
pm2 delete ktltc
pm2 start npm --name "ktltc" -- start
pm2 save
